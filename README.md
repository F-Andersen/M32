# Автоматизована система обліку продажу меблів

Настільний застосунок для невеликого меблевого магазину, який дозволяє вести
каталог товарів, реєструвати продажі, контролювати залишки та формувати звіти.
Програма написана мовою **Python 3** з використанням **Tkinter** для GUI та
**SQLite** як вбудованої реляційної бази даних.

## Основні можливості

- **Каталог меблів**
  - додавання, редагування та видалення товарів;
  - зберігання назви, категорії, матеріалу, кольору, розмірів (Ш×В×Г),
    базової ціни та залишку на складі;
  - фільтрація за частиною назви, назвою категорії та діапазоном цін;
  - швидке заповнення каталогу тестовими даними (≈50 товарів).

- **Оформлення продажів**
  - вибір товару з каталогу через випадаючий список;
  - введення кількості, ціни продажу та знижки (%);
  - автоматичний розрахунок суми продажу;
  - перевірка наявності товару на складі, заборона «мінусових» залишків;
  - запис операції у журнал продажів і оновлення залишків.

- **Звіти**
  - виручка за днями (агрегований список дата → сума);
  - актуальні залишки товарів на складі;
  - експорт даних виручки та залишків у формат **CSV**
    для подальшої обробки в Excel / LibreOffice.

- **Журнал подій**
  - фіксація основних дій (додавання/редагування товарів, продажі);
  - зручний перегляд історії роботи з програмою.

---

## Використані технології

- **Мова програмування:** Python 3.x  
- **GUI:** Tkinter / ttk (стандартна бібліотека)  
- **База даних:** SQLite (через модуль `sqlite3`)  
- **Робота з файлами та експорт:** модуль `csv`, `pathlib`, `datetime`

## Структура проєкту

Приблизна структура файлів (може відрізнятися залежно від  реалізу):

```text
project_root/
├─ main.py              # Точка входу, запуск GUI та ініціалізація БД
├─ db_layer.py          # Функції роботи з SQLite, init_db(), виконання SQL-запитів
├─ services.py          # Бізнес-логіка (товари, продажі, звіти, журнал)
├─ ui_main.py           # Побудова інтерфейсу Tkinter (вкладки, форми, таблиці)
├─ furniture.db         # Файл бази даних SQLite (створюється автоматично)
├─ data/
│  ├─ exports/          # CSV-файли з експортованими звітами
│  └─ test_data.json    # (опціонально) опис тестових товарів
└─ README.md            # Цей файл
